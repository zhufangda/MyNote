stage: 将工作区的文件保存至暂存区(staging area)	git -add
commit：将暂存区的文件提交到仓库中	git -commit "Message"
Add branch:增加新的分支
Check-out:将选择分支设置为当前分支
.gitignore:记录要忽略的文件
merge:将选择分支与当前分支合并


先将文件推送到暂存器，再将文件推送到仓库。之后修改新文件，我们将能看到工作区与暂存区文件的对比。
单击log，能看到仓库文件的版本库。当在主窗选择根文件，log则显示整个文件的版本，标题栏显示为"Log for.",其中.代表根文件夹。由绿色三角指示的分支的当前分支。
